## Task: Implement a sliding-window rate limiter

A teammate left behind an incomplete module for rate-limiting API calls per user. The interface is sketched out; your job is to write the body of each method.

### Background

Our API gateway enforces per-user limits: at most `max_calls` requests in any rolling window of `window_seconds` seconds. The decision must be made efficiently and memory per user must stay bounded — once a timestamp falls outside the window it can be discarded.

### Skeleton to complete

```python
from __future__ import annotations
import time
from collections import deque
from dataclasses import dataclass, field
from typing import Optional


@dataclass
class RateLimiter:
    """Sliding-window rate limiter for a single user.

    Attributes:
        max_calls: Maximum number of calls allowed in any window of
            `window_seconds` length.
        window_seconds: Width of the rolling window in seconds.
    """
    max_calls: int
    window_seconds: float
    _timestamps: deque[float] = field(default_factory=deque, init=False, repr=False)

    def is_allowed(self, now: Optional[float] = None) -> bool:
        """Return True if a new request at time `now` is within the rate limit
        and record the timestamp.

        If `now` is None, use the current wall-clock time.

        Rejected calls are NOT recorded: a client that is rate-limited should
        be able to retry once the window clears without being penalised for
        failed attempts.

        Must purge expired timestamps before checking so memory stays bounded.
        """
        ...

    def remaining(self, now: Optional[float] = None) -> int:
        """Return how many more calls are allowed in the current window.

        Must not modify internal state.
        """
        ...

    def reset_at(self, now: Optional[float] = None) -> Optional[float]:
        """Return the epoch timestamp at which the oldest in-window call expires.

        Returns None when the limiter is not currently saturated (i.e. when
        remaining() > 0).  Must not modify internal state.
        """
        ...
```

### Behavioural requirements

1. Thread safety is not required; callers handle locking externally.
2. `remaining` and `reset_at` must never add or remove timestamps; they are
   read-only views of the current state.
3. All three methods accept an optional `now` parameter so tests can inject a
   fake clock without monkey-patching `time.time`.
4. `reset_at` returns the expiry time of the *oldest* call in the window, not a
   duration from now, and only when `remaining() == 0`.

### Expected behaviour (executable examples)

```python
rl = RateLimiter(max_calls=3, window_seconds=1.0)

assert rl.is_allowed(now=0.0) is True    # 1st call — allowed
assert rl.is_allowed(now=0.3) is True    # 2nd call — allowed
assert rl.is_allowed(now=0.6) is True    # 3rd call — allowed
assert rl.is_allowed(now=0.9) is False   # 4th in <1 s window — rejected, not recorded
assert rl.remaining(now=0.9) == 0
assert rl.reset_at(now=0.9) == 1.0      # oldest call (t=0.0) expires at t=1.0

# Window rolls: call at t=0.0 has expired by t=1.05
assert rl.is_allowed(now=1.05) is True   # window now contains [0.3, 0.6, 1.05]
assert rl.remaining(now=1.05) == 0
assert rl.reset_at(now=1.05) == 1.3     # oldest surviving call (t=0.3) expires at t=1.3

# Edge case: empty limiter
fresh = RateLimiter(max_calls=5, window_seconds=60.0)
assert fresh.remaining(now=0.0) == 5
assert fresh.reset_at(now=0.0) is None
```
